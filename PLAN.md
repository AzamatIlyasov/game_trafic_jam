# Симулятор светофоров и пробок — план

## Контекст

Браузерная игра-симулятор дорожного движения, играбельная на телефоне (iPhone Safari) и десктопе.
Цель — интерактивно показать, как из простого поведения водителей рождаются пробки, в том числе
**«фантомная» пробка** (stop-and-go волна): затор, возникающий «из ниоткуда» без аварии и без
узкого места, только за счёт динамики следования за лидером.

Игрок может настраивать: число светофоров/перекрёстков, дистанцию между перекрёстками, тайминги
светофоров, плотность потока и параметры водителя — и наблюдать, как меняется картина движения.

**Стек:** Vite + TypeScript, HTML5 Canvas 2D, без UI-фреймворка. Чистая статика, деплой в GitHub Pages.
**Репозиторий:** `azamatilyasov/game_trafic_jam`, ветка `claude/traffic-light-simulator-game-btyh3v`.

## Модель симуляции: Intelligent Driver Model (IDM)

Каждый автомобиль `i` едет по 1D-координате вдоль дороги (арк-длина). Лидер — машина `i+1`.

```
s   = x_lead - x - L_lead                        // чистый зазор до лидера
Δv  = v - v_lead                                 // скорость сближения
s*  = s0 + max(0, v·T + v·Δv / (2·√(a·b)))       // желаемый зазор
dv/dt = a · [ 1 - (v/v0)^δ - (s*/s)² ]           // ускорение (ОДУ)
```

**Параметры по умолчанию** (метрические, легковой авто):

| Символ | Смысл | Значение |
|---|---|---|
| `v0` | желаемая скорость | 30 м/с (шоссе) / ~14 м/с (город) — слайдер |
| `T` | безопасный интервал | 1.5 с — **главная «ручка» пробок** |
| `a` | макс. ускорение | 1.5 м/с² |
| `b` | комфортное торможение | 2.0 м/с² |
| `δ` | экспонента | 4 |
| `s0` | мин. зазор в заторе | 2.0 м |
| `L` | длина машины | 5.0 м |
| `bMax` | предел торможения | 9.0 м/с² |

**Интегрирование:** полуявный (баллистический) Эйлер, фиксированный шаг `dt = 0.04 с`.
Все ускорения считаются из **старого** состояния, затем применяются (без мутации на лету).

```
a_i = clamp(idmAccel(...), -bMax, a)
v_i' = max(0, v_i + a_i·dt)
x_i' = x_i + v_i·dt + 0.5·a_i·dt²
```

### Как рождается «фантомная» пробка и как её вызвать

IDM **строчно неустойчив** выше критической плотности: малое возмущение скорости усиливается,
распространяясь вверх по потоку, и превращается в устойчивую stop-and-go волну, бегущую **назад**
(классический кольцевой эксперимент Сугиямы). Два триггера в игре:

1. **Плотность** — слайдер добавляет/убирает машины на кольце. Выше критической плотности
   равномерный поток сам распадается на волны.
2. **Кнопка «Вызвать пробку»** — одна машина на 1–2 с резко тормозит (`perturbId`+`perturbTimer`).
   При низкой плотности возмущение затухает; при высокой — разрастается в бегущую волну. Это главный демо-момент.

### Светофор как граничное условие

Красный/жёлтый = **виртуальный неподвижный лидер** на стоп-линии для ближайшей подъезжающей машины:
`s = стоп-линия − x`, `Δv = v`, `v_lead = 0` → машина плавно тормозит до остановки. Зелёный убирает
виртуального лидера → очередь разъезжается волной разгона. Лидер машины = **ближайший из**
{реальная машина впереди, активный красный}. Жёлтый: если машина уже не успевает остановиться
(`v²/(2b)` больше зазора) — проезжает (обработка «дилемма-зоны»).

## Рендер и игровой цикл

- **Canvas 2D**, два слоя: `#road` (статичный фон — дорога, разметка, перекрёстки; перерисовка при
  изменении конфигурации) и `#sim` (машины, светофоры, HUD — каждый кадр). Третий слой `#spacetime`
  для диаграммы пространство-время.
- **DPR-масштабирование** (cap 2) для чёткости на iPhone.
- **Фиксированный шаг + аккумулятор**, рендер отделён от физики:

```
acc += frameTime · speedMultiplier;
while (acc >= DT && steps < MAX_STEPS) { sim.step(DT); acc -= DT; steps++; }
renderer.draw(sim);
```

Множитель скорости масштабирует время симуляции (0.25×–8×), не `dt`. Пауза = не вызывать `step`.

**Геометрия дороги** (интерфейс `RoadGeometry`: `toXY(dist)`, `length`, `wrap`, `isRing`):
- **Кольцо** — периодическая граница, эталон фантомной пробки.
- **Магистраль** — прямые отрезки между перекрёстками, спавн/деспавн на концах.
- **Сетка** — несколько дорог с общими узлами-перекрёстками и координацией светофоров.

```
game_trafic_jam/
├── index.html                 # canvas-слои + панель управления
├── vite.config.ts             # base:'./' для Pages
├── package.json, tsconfig.json
├── src/
│   ├── main.ts                # bootstrap, владеет RAF-циклом
│   ├── loop.ts                # GameLoop: фикс. шаг + аккумулятор
│   ├── sim/
│   │   ├── types.ts           # SimConfig, IDMParams, Vehicle, TrafficLight, SimState
│   │   ├── idm.ts             # idmAccel(v,s,dv,params) + пресеты
│   │   ├── vehicle.ts         # тип Vehicle + фабрика
│   │   ├── geometry.ts        # RingRoad / LinearRoad / GridRoad
│   │   ├── trafficLight.ts    # автомат состояний + virtualLeader()
│   │   └── simulation.ts      # состояние, разрешение лидера, step(dt), spawn, perturb
│   ├── render/
│   │   ├── roadRenderer.ts    # статичный фон
│   │   ├── simRenderer.ts     # машины + светофоры + HUD за кадр
│   │   └── spacetime.ts       # диаграмма пространство-время
│   ├── ui/
│   │   ├── controls.ts        # слайдеры/кнопки → конфиг
│   │   └── hud.ts             # живые метрики
│   ├── metrics/metrics.ts     # ср. скорость, пропускная способность, время ожидания
│   └── styles.css             # touch-friendly, крупные тап-цели, safe-area
└── .github/workflows/deploy.yml  # автодеплой в GitHub Pages
```

Ключевой инвариант: `simulation.ts` держит массив машин **отсортированным** по позиции (вставками, O(n)),
рендереры — **только читатели** состояния, `geometry.ts` — единственное место, знающее про 2D.

## UI управления (touch-friendly)

Сворачиваемая нижняя панель, крупные `<input type="range">` с живыми значениями:

| Контрол | Диапазон |
|---|---|
| Перекрёстки (кол-во) | 0–12 |
| Дистанция между ними (м) | 50–500 |
| Длина кольца (м) | 200–2000 |
| Плотность / число машин | 0–200 |
| Желаемая скорость `v0` | 5–35 м/с |
| Интервал `T` | 0.6–2.5 с |
| Зелёный / Красный (с) | 5–60 |
| Смещение «зелёной волны» | 0–цикл |
| Интенсивность спавна | 0–2 маш/с |

Кнопки: **Пуск/Пауза**, **Сброс**, **Вызвать пробку**, **Скорость ×**, **Режим** (Кольцо/Магистраль/Сетка),
переключатель **Диаграммы пространство-время**.

## Метрики и визуализация

- **Средняя скорость** (км/ч), **пропускная способность** (маш/ч через детектор), **суммарное время
  ожидания** (Σ dt при `v < 0.5 м/с`), **плотность** (маш/км).
- **Диаграмма пространство-время** — главная визуализация: ось X = позиция, ось Y = время (вниз),
  цвет = локальная скорость (зелёный→красный). Фантомная пробка выглядит как **красные диагональные
  полосы, наклонённые вверх по потоку** — безошибочная подпись бегущей волны.

## Дополнительные идеи для игры

1. **Режим «Зелёная волна»** — подбирать смещения фаз светофоров, чтобы поток проходил магистраль без
   остановок; метрики показывают выигрыш.
2. **Умный светофор (AI) vs фиксированный** — адаптивная фаза по длине очереди, сравнение метрик бок о бок.
3. **Челлендж/пазл-уровни** — «пропусти N машин за T», «убери фантомную пробку одним светофором»,
   очки за минимизацию суммарного ожидания.
4. **Час пик** — переменная интенсивность спавна по «времени суток».
5. **Спецтранспорт** — скорая с приоритетом, «зелёный коридор».
6. **Многополосность + перестроения** — обгон, эффект «зиппера» при сужении.
7. **Кольцевая развязка vs светофор** — сравнить пропускную способность одного узла.
8. **Фундаментальная диаграмма** (плотность–поток) в реальном времени — образовательный график.
9. **Пресеты и шаринг** — «Кольцо Сугиямы», «Час пик»; конфиг в URL и `localStorage`.
10. **Тепловая карта заторов** поверх дороги.

## Поэтапный план

- **Фаза 0 — Каркас:** Vite+TS, два canvas-слоя, DPR, пустой GameLoop с фикс. шагом, счётчик FPS.
  Проверка 60 fps на iPhone.
- **Фаза 1 — MVP: кольцо + IDM + фантомные пробки:** `idm.ts`, `geometry.ts`(RingRoad),
  `simulation.ts`, рендер машин по скорости, слайдеры (плотность, `v0`, `T`), кнопка «Вызвать пробку»,
  **диаграмма пространство-время**. Приёмка: рост плотности/тормозной импульс рождает устойчивую
  бегущую назад волну.
- **Фаза 2 — Перекрёстки и светофоры:** `trafficLight.ts`+`virtualLeader()`, `LinearRoad`, спавн/деспавн,
  разрешение лидера с учётом красного, слайдеры числа перекрёстков/дистанции/циклов, переключатель режима.
  Приёмка: машины плавно тормозят у красного, копят очередь, разъезжаются на зелёный.
- **Фаза 3 — Метрики, сетка, зелёная волна:** `metrics.ts`+`hud.ts`, `GridRoad`, слайдер смещения фаз.
- **Фаза 4 — Полировка:** пресеты, шаринг конфига через URL, `localStorage`, safe-area, спрайты.
- **Деплой:** `.github/workflows/deploy.yml` собирает `vite build` и публикует `dist/` в GitHub Pages
  (`base:'./'` для относительных путей).

## Проверка

- Локально: `npm install && npm run dev`, открыть в браузере/эмуляции iPhone, проверить 60 fps.
- Фантомная пробка: поднять плотность до критической ИЛИ нажать «Вызвать пробку» на кольце → на
  диаграмме пространство-время появляются красные полосы, наклонённые вверх по потоку.
- Светофоры: включить магистраль, задать перекрёстки → очереди у красного, разъезд на зелёный.
- После деплоя: открыть Pages-ссылку на реальном iPhone, проверить тач-управление и safe-area.
